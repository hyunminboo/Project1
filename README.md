## 1.프로젝트 개요

*이 프로젝트를 선택한 이유는 평소에 자주 즐겨보는 게임, 애니메이션, 영화를 다른 사람들과 공유하고 싶었기 때문이다.
또한 웹페이지를 직접 구현해보며 HTML과 CSS를 활용한 웹 개발 연습을 하고자 이 프로젝트를 진행하게 되었다.
관심 있는 콘텐츠를 주제로 삼아 개발함으로써, 학습에 대한 흥미를 높이고 연습에 도움이 될 수 있도록 방향을 잡았다.

## 2.HTML작성 사 신경 쓴 점

*웹페이지를 사용하는 사용자가 한눈에 내용을 파악할 수 있도록 구조를 최대한 직관적으로 구성하였다.
불필요한 요소는 배제하고, 필요한 정보만 명확하게 보이도록 HTML을작성하여 가독성과 사용성을 높이는 데 집중했다.

## 3.작성 시 신경 쓴 점

*전체적인 디자인은 과하지 않으면서도 콘텐츠가 잘 드러나도록 구성하였다.
레이아웃과 여백을 적절히 사용해 시각적인 답답함을 줄였고, 색상과 폰트 크기를 통일하여 일관성 있는 화면을 유지하도록 했다.
또한 각 요소의 역할이 명확하게 보이도록 스타일을 적용해 사용자 경험을 고려했다.

## 4.구현 중 문제와 해결

*웹페이지를 구현하는 과정에서 레이아웃이 의도한 대로 배치되지 않는 문제가 발생했다.
이 문제는 HTML 구조를 다시 정리하고, CSS에서 display, position, margin, padding 값을 조정하면서 해결하였다.

## 5.개선할 점

*프레임 구성과 배치를 어떻게해야 유저 친화적인 디자인이 될지 감을 잡지 못했다.
설계할때 제대로 틀을 잡지 않고 시작해 코드 작성중 해매는 경우가 너무나도 많이 발생 하였다. 
확실한 기준점이 없으니 만들면서 계속 바뀌면서 이도저도 아닌 상황이 발생하였다.
이런부분에 있어서 확실한 설계와 디자인을 먼저 생각하고 시작해야 한다는것을 느꼇다.
